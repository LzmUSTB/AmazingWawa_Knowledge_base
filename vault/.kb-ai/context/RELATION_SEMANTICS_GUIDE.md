# Relation Semantics Guide

Relation types are frozen:

```text
contains
depends-on
used-in
compares-with
```

Do not create new relation types.

## contains

Meaning:
- hierarchy / containment,
- usually generated by `parentId`,
- not manually added with `add_edge`.

Example:

```yaml
parentId: computer-graphics
node.id: curl-noise
```

This creates:

```text
computer-graphics contains curl-noise
```

## depends-on

Meaning:

```text
A depends-on B
```

Understanding B helps understand A.

Good examples:
- `curl-noise depends-on divergence-and-curl`
- `pressure-projection depends-on divergence`
- `pbr-material depends-on brdf`

Bad examples:
- `divergence-and-curl depends-on curl-noise` if divergence/curl is more foundational.
- `real-time-vfx depends-on curl-noise` if Curl Noise is only one technique used in VFX.

Direction matters.

## used-in

Meaning:

```text
A used-in B
```

A is a technique, component, or concept used inside B.

Good examples:
- `curl-noise used-in real-time-vfx`
- `normal-map used-in pbr-material`
- `prefix-sum used-in gpu-particle-sorting`

Bad examples:
- `real-time-vfx used-in curl-noise`

## compares-with

Meaning:
- A and B are alternatives, contrasts, or related concepts that answer similar questions.
- Stored once, displayed as bidirectional.

Good examples:
- `curl-noise compares-with flow-map`
- `sph compares-with mls-mpm`
- `ray-marching compares-with rasterization`

## Relation quality rules

A relation should be created only when it helps future navigation or learning.

Do not create edges just because two concepts appeared in the same article.

Every non-contains relation should be explainable in one sentence:

```text
Curl Noise depends-on Divergence and Curl because its vector field construction relies on the idea of local rotation and near-zero divergence.
```

## Relation plan

Every package should include relation rationale in:

```text
review/relation-rationale.md
```

Recommended format:

```yaml
relations:
  - from: curl-noise
    relation: depends-on
    to: divergence-and-curl
    rationale: Curl Noise is defined through a curl-like vector field derived from scalar noise.
```
