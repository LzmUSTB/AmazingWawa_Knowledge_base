# 渲染管线

::concept-card
title: 一句话理解
content: 渲染管线就是把三维世界一步步转换成屏幕像素的过程。
::

## 一句话定义223

渲染管线是把三维场景转换成二维图像的一系列处理阶段。

## 它解决什么问题？

它为实时渲染提供稳定的处理顺序：收集场景数据、变换顶点、生成片元、计算颜色、执行后处理，并最终输出到屏幕。

## 核心直觉

把每个阶段看成一个有明确输入和输出的模块。知识图谱负责表示这些模块之间的关系，笔记负责解释每个模块内部的含义。

::process-flow
title: 基础渲染流程
steps:
  - 应用阶段
  - 顶点处理
  - 光栅化
  - 片元处理
  - 后处理
::

## 正式解释

在传统实时渲染中，CPU 侧通常准备场景数据和渲染命令，GPU 侧执行顶点处理、光栅化、片元处理等阶段。不同图形 API 和引擎会封装这些步骤，但核心结构相对稳定。

## 最小例子

```txt
Scene Data -> Vertex Processing -> Rasterization -> Fragment Shading -> Post Process -> Display
```

## 常见误区

- 把渲染管线理解成固定不可变的流程。
- 忽略应用阶段对最终渲染结果的影响。
- 把 Shader 只理解成“材质”，而不是可编程处理阶段。
