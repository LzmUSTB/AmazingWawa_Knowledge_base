# 程序化 VFX 纹理

:::concept-card
title: 程序化 VFX 纹理
summary: 用噪声、图像处理、向量场和材质网络生成可控的实时特效纹理资产。
why: 游戏特效经常需要烟、火、能量、液体、魔法轨迹、冲击波等有机变化；程序化方法能在较低成本下快速迭代大量变体。
key_intuition: 不直接模拟真实物理，而是把“看起来像运动”的结构拆成噪声、扭曲、混合、遮罩、动画和贴图导出。
:::

## 定义

程序化 VFX 纹理是指通过算法或节点图生成的特效纹理。它可以作为材质的颜色、透明度、法线、UV offset、flipbook 帧或遮罩使用。

这类知识点关注的是 **资产生成逻辑**，而不是完整的物理模拟。Curl noise 就属于其中一种典型方法：它用向量场扭曲图案，制造类似流体旋涡的视觉结果。

## 解决的问题

手绘或逐帧制作有机纹理成本高，且难以系统控制尺度、速度、方向和层次。程序化纹理把这些因素变成参数，例如 noise scale、octaves、warp intensity、mask、blend mode、tiling mode 和 time input。

## 最小例子

1. 生成一张基础噪声或几何图案。
2. 用另一张噪声或向量场扭曲它。
3. 通过 remap / levels / gradient map 控制颜色和对比度。
4. 输出为静态贴图、循环贴图或 flipbook。
5. 在游戏 shader 中作为 VFX 材质输入使用。

:::compare-table
columns: [方法, 适合用途, 主要风险]
rows:
  手绘纹理: [明确美术风格、关键帧式图案, 变体成本高]
  程序化纹理: [大量迭代、可参数化控制, 容易产生机械感]
  完整流体模拟: [物理可信度高、真实流动, 计算与制作成本高]
:::

## 常见误区

- **把程序化等同于真实物理。** 很多实时 VFX 只追求视觉可信，不追求物理严格。
- **只看生成器，不看最终 shader。** 同一张贴图在不同材质、混合模式和粒子系统中效果差异很大。
- **忽略导出约束。** 游戏中常要考虑分辨率、压缩、帧数、循环、tiling 和显存。
